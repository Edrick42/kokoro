# Kokoro — DALL·E 3 Sprite Prompts (Pixel Art Version)
# 64x64 pixel art — optimized for in-game color tinting via genome hue gene

## HOW TO USE

1. Start a NEW ChatGPT conversation
2. Send Prompt 0 (Idle) first — no image attached
3. From Prompt 1 onwards: ATTACH the approved Idle image before sending
4. If the character drifts, start a new conversation attaching the Idle image first
5. After all moods are approved, generate size variants (section at the end)

## CRITICAL DESIGN RULES (read before generating)

- True pixel art style — hard pixel edges, NO anti-aliasing whatsoever
- Limited color palette — maximum 6 colors total:
  flat blue body, black outline, white eye whites, black pupils,
  light blue for inner ear, one blush pink for cheeks (optional)
- FLAT body color only — NO shading, NO gradients, NO dithering on the body
  Reason: the game applies color tinting in code from the genome hue gene.
  Any baked-in shading will break when tinted.
- Bold 1-pixel black outline everywhere, fully connected, no gaps
- Canvas: 64x64 pixels, pure white or transparent background
- Style reference: classic Tamagotchi LCD art, Neopets pixel pets,
  early Game Boy Color sprites — chunky, readable, expressive at tiny size

## IMPORTANT: DALL·E LIMITATION WITH PIXEL ART

DALL·E 3 sometimes struggles with true pixel art and adds smoothing.
If the result looks smooth/blurry instead of pixelated:

Option A — add this to your prompt:
"Extremely pixelated, each pixel is a visible hard square, like a
Game Boy Color game screenshot, no smoothing whatsoever, 8-bit style"

Option B — generate with the flat style prompts instead, then
convert to pixel art manually using Aseprite or Photopea:
  Photopea: Image → Image Size → 64x64 → Nearest Neighbor
  Then scale back up: Image → Image Size → 512x512 → Nearest Neighbor

Option B gives perfect pixel art every time and is worth the
extra 2 minutes per sprite.
```
