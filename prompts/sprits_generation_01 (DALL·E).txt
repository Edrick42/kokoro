## HOW THE MODULAR SYSTEM WORKS

The game engine assembles the creature by layering parts:

```
Z-depth layering (back to front):
  -0.1  ears (behind body)
   0.0  body (center)
   1.0  eyes, mouth (on top of body)
   2.0  accessories/effects (overlays)
```

- Body, ears: TINTED by the genome hue gene (must be flat color)
- Eyes, mouth: NOT tinted (keep original colors)
- Accessories: NOT tinted (overlays on top)

When mood changes, ONLY eyes and mouth swap sprites.
Body and ears stay the same across all moods.

## HOW TO USE THESE PROMPTS

1. Start a NEW ChatGPT conversation
2. Generate the BODY first (Prompt 0) — this is the base reference
3. Then generate ears, eyes, and mouth separately
4. For mood-reactive parts (eyes, mouth), generate each mood variant
5. Keep everything in the SAME ChatGPT conversation for consistency
6. If the style drifts, start fresh and attach the body as reference

---

## CRITICAL DESIGN RULES

- True pixel art style — hard pixel edges, NO anti-aliasing
- Limited palette — max 6 colors: flat blue body, black outline,
  white eye whites, black pupils, light blue inner ear, optional blush pink
- FLAT body color — NO shading, NO gradients, NO dithering on body.
  Reason: the game applies color tinting from the genome hue gene.
  Any baked-in shading breaks when tinted to other colors.
- Bold 1-pixel black outline, fully connected, no gaps
- Canvas: 64x64 pixels, transparent background
- Each part must be on its OWN transparent canvas (not the full character)
- Style reference: classic Tamagotchi LCD art, Neopets pixel pets,
  early Game Boy Color sprites

## PROMPT 0 — Body (generate first, no image attached)

The body is the largest part and sets the style for everything else.
It should NOT include eyes or mouth — those are separate parts.

```
True pixel art sprite, 64x64 pixel canvas, hard pixel edges,
zero anti-aliasing, zero sub-pixel blending, transparent background.

A round chubby blob body for a virtual pet creature called Kobara.
This is ONLY the body — no eyes, no mouth, no facial features.

- Shape: round chubby blob with tiny stubby pixel arms and legs,
  small rounded pixel tail on the right side
- NO ears (ears are separate sprites)
- NO eyes, NO mouth, NO nose (facial features are separate sprites)
- The face area should be blank/flat — just the body color
- Outline: solid 1-pixel black border, fully connected, no gaps
- Fill: flat solid pastel blue (#A8D8EA), completely uniform,
  NO gradients, NO shading, NO dithering
- Background: fully transparent

Upscale to 512x512 using nearest-neighbor scaling (pixel-perfect,
no smoothing — each pixel becomes an 8x8 block).

Negative: no facial features, no eyes, no mouth, no anti-aliasing,
no smooth edges, no gradients, no shading, no dithering, no blur,
no watermark, no text, no background elements.
```

## COLOR TINT TESTING

After generating the body, test that tinting works by recoloring:

### Coral pink variant
```
Take the Kobara body pixel art and recolor all blue pixels (#A8D8EA)
to flat coral pink (#F0997B). Keep everything else pixel-perfect
identical — same outline, same composition. No smoothing.
```

### Golden yellow variant
```
Take the Kobara body pixel art and recolor all blue pixels (#A8D8EA)
to flat golden yellow (#EF9F27). Keep everything else identical.
```

If the body looks good in both colors, the flat fill is correct
and in-game tinting will work properly.

## DALL·E PIXEL ART TIPS

- DALL·E 3 sometimes adds smoothing. If results look blurry:
  Add "Extremely pixelated, each pixel is a visible hard square,
  like a Game Boy Color screenshot, 8-bit style" to the prompt.

- Fallback method (guaranteed pixel art):
  1. Generate with the flat style prompts
  2. Open in Photopea (free, online)
  3. Image → Image Size → target size → Nearest Neighbor
  4. Scale back up → Nearest Neighbor

- Keep ALL generations in ONE ChatGPT session for style consistency.
- If character drifts, start fresh and attach the body as reference.

## FUTURE SPECIES NOTES

When adding a new species (e.g. Lumini bird), follow this pattern:

1. Generate the BODY first (defines the species visual identity)
2. Generate species-specific parts (wings, beak, tail, etc.)
3. Generate mood variants ONLY for mood-reactive parts
4. Static parts (body, wings at rest) only need the idle variant
5. Use the same color rules: flat fill for tinted parts, detailed for others
6. Drop files into `assets/sprites/{species_name}/`
7. Register the species template in `src/systems/body_parts.rs`
